3. Keep under 500 lines (use progressive disclosure if needed)
4. Focus on specialized knowledge Claude doesn't already have
5. Include concrete examples, not abstract theory

**Examples (code files):**
1. Must be runnable or clearly marked as pseudocode
2. Include comments explaining key decisions
3. List dependencies at the top of the file
4. Follow language-specific best practices
5. Keep focused on demonstrating one pattern well

**Scripts (*.py, *.sh):**
1. Include shebang and usage documentation
2. Add error handling and validation
3. Make idempotent where possible
4. Accept parameters via CLI arguments
5. Output progress and results clearly

**Assets (JSON, templates, etc.):**
1. Include schema documentation
2. Provide realistic example data

## Appendix: File Naming Conventions

### References
- Use kebab-case: `pagination-patterns.md`
- Be descriptive: `oauth21-guide.md` not `auth.md`
- Group related: `orms-python.md`, `orms-typescript.md`, `orms-rust.md`

### Examples
- Include framework: `tanstack-basic.tsx`, `fastapi-streaming.py`
- Describe pattern: `streaming-chat.tsx`, `virtual-scrolling-grid.tsx`
- Use extensions: `.tsx`, `.py`, `.rs`, `.go`

### Scripts
- Use verbs: `generate_tokens.py`, `validate_schema.py`
- Include extension: `.py`, `.sh`
- Make executable: `chmod +x scripts/*.sh`

### Assets
- Descriptive names: `grid-layouts.json`, `theme-tokens.json`
- Include format: `.json`, `.yaml`, `.svg`
- Version if needed: `schema-v1.json`
